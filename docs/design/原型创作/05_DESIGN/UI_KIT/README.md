## 2. UI Kit 的目标

UI Kit 不是为了“收集样式”，而是为了让平台的设计和开发都保持一致。

本项目之所以需要 UI Kit，是因为它有明显的平台属性，页面类型多、状态多、角色多：

- 孩子端页面
- 家长端页面
- 作品页
- 候选状态页
- 分享推广页
- 后台页

如果没有组件库，后面很容易出现：

- 每页按钮长得不一样
- 候选标签每页颜色不同
- 作品卡片每页结构不同
- 奖励状态每页展示不同

---

## 3. UI Kit 的设计原则

### 3.1 先搭基础组件，再搭业务组件
不要一上来就做复杂大组件。

### 3.2 先保住作品卡和状态标签
因为这个平台最核心的是作品与状态。

### 3.3 孩子端和家长端应风格统一，但层级不同
不能做成两个完全不同的产品。

### 3.4 候选状态和奖励状态必须全平台统一
这是后续最容易乱的地方。

---

## 4. UI Kit 建议分层

建议把组件分成三层：

---

## 4.1 基础组件层

### 目标
搭建最底层通用组件。

### 建议组件
- 按钮 Button
- 输入框 Input
- 选择器 Select
- 标签 Tag
- 卡片 Card
- 弹窗 Modal
- 空状态 Empty State
- 分段切换 Tabs
- 状态提示 Toast / Message

---

## 4.2 页面通用组件层

### 目标
把多个页面都会反复使用的组件统一出来。

### 建议组件
- 页面标题栏
- 模块入口卡
- 作品卡
- 状态卡
- 进度卡
- 说明卡
- 奖励记录卡
- 邀请记录卡

---

## 4.3 业务组件层

### 目标
为本平台独有的业务逻辑提供专属组件。

### 建议组件
- 原型标签组
- 图形标签组
- 动作标签组
- 候选状态条
- 展示状态条
- 作品说明卡
- 家长待办卡
- 分享推广总览卡

---

## 5. 首批必须优先做的组件

如果资源有限，建议先做这 10 个组件：

1. 主按钮 Button Primary  
2. 次按钮 Button Secondary  
3. 标签 Tag  
4. 卡片 Card  
5. 模块入口卡 Module Card  
6. 作品卡 Work Card  
7. 状态标签 Status Tag  
8. 空状态 Empty State  
9. 页面标题栏 Page Header  
10. 家长待办卡 Parent Task Card  

---

## 6. 状态相关组件必须统一

这个平台有很多状态，建议都从组件库里统一管理。

### 6.1 候选状态标签
- 文创候选
- 存证候选
- 确权候选

### 6.2 展示状态标签
- private
- family
- public_candidate
- public

### 6.3 奖励状态标签
- pending
- settled
- invalid
- reversed

### 6.4 作品状态标签
- uploaded
- reviewed
- featured
- archived

这些一定不要每页自己做。

---

## 7. 作品卡组件必须重点设计

因为这个平台本质上是作品平台。

建议作品卡至少有两个版本：

### 7.1 小卡版
用于：
- 我的展馆
- 最近作品
- 家长中心作品列表

### 7.2 大卡版
用于：
- 作品详情前置推荐
- 精选区
- 首页重点作品展示

### 作品卡建议包含
- 缩略图
- 标题
- 模块标签
- 状态标签
- 时间信息

---

## 8. 模块组件建议

### 模块入口卡
适用于：
- 首页
- 训练中心

建议包含：
- 模块图标
- 模块名称
- 模块说明
- 当前进度
- 进入按钮

### 模块差异化建议
- 模块A：更偏组合与装饰感
- 模块B：更偏结构与生成感

但整体风格要统一，不能像两个 App。

---

## 9. 分享推广中心组件建议

因为这是后续很容易乱掉的地方，建议单独抽组件。

### 建议组件
- 邀请总览卡
- 邀请入口卡
- 奖励规则说明卡
- 邀请记录卡
- 奖励记录卡

这样后面无论做页面还是改版，都不会散。

## 12. 一句话总结

本平台的 UI Kit，不是为了让页面“看起来差不多”，而是为了让：

**作品、状态、候选、奖励、家长路径这些关键元素，在全平台都拥有统一而可信的视觉语言。**
