## 2. 用户流程图的目标

用户流程图不是为了好看，而是为了把平台最关键的行为路径画清楚。

本项目的流程图尤其重要，因为它不是一个单点工具，而是一个包含：

- 双训练模块
- 作品上传
- 作品展示
- 家长管理
- 候选升级
- 分享推广

的一体化平台。

如果没有流程图，后面很容易出现：

- 训练链清楚，但作品链断掉
- 家长中心有页面，但不在主路径里
- 候选系统存在，但用户不知道从哪里进
- 分享推广中心挂在边上，不进入主流程

---

## 3. 本项目用户流程图的设计原则

### 3.1 先画主链，再画支链
优先把最关键的用户路径画清楚，不要一开始画太多边角流程。

### 3.2 先保住“作品生成主链”
本项目最重要的流程不是“点页面”，而是：

**训练 → 创作 → 上传 → 作品 → 展示 → 家长管理**

### 3.3 家长路径必须单独画
家长不是普通查看者，而是：
- 展示权限管理者
- 候选状态选择者
- 分享推广发起者

### 3.4 分享推广必须画成独立闭环
因为这条路径和普通内容平台不一样，它不只是“分享一个链接”，而是：

**邀请 → 注册 → 激活 → 奖励 → 后续净收益**

## 4. 建议优先绘制的用户流程

建议按以下顺序出图。

---

## 第一批：核心主链（最优先）

### 4.1 孩子训练主链
首页  
→ 训练中心  
→ 选择模块  
→ 进入任务  
→ 打印底稿  
→ 纸上创作  
→ 上传作品  
→ 作品详情  
→ 我的展馆

---

### 4.2 作品生成主链
任务页  
→ 打印页  
→ 纸上完成  
→ 上传页  
→ 上传成功页  
→ 作品详情页  
→ 展馆展示

---

### 4.3 家长作品管理主链
家长中心  
→ 查看孩子作品  
→ 修改标题 / 描述  
→ 设置展示状态  
→ 设置候选状态  
→ 返回作品详情页查看结果

---

## 第二批：成长与候选主链

### 4.4 成长档案路径
我的展馆  
→ 成长档案页  
→ 查看时间流  
→ 查看模块分布  
→ 查看系列作品

---

### 4.5 候选升级路径
作品详情页  
→ 家长中心  
→ 设置文创候选 / 存证候选 / 确权候选  
→ 候选状态显示  
→ 候选页入口（占位）

---

## 第三批：分享推广主链

### 4.6 家长分享推广主链
家长中心  
→ 分享推广中心  
→ 生成邀请码 / 邀请链接  
→ 被邀请人注册  
→ 被邀请人激活  
→ 奖励记录生成  
→ 家长查看奖励记录

---

## 第四批：后续扩展链（可后画）

### 4.7 家庭共享链
作品详情  
→ 家长选择 family 可见  
→ 家庭共享区

### 4.8 公共展示链
作品详情  
→ 家长申请公开候选  
→ 后台审核  
→ 公共展示区

### 4.9 文创转化链
作品详情  
→ 家长加入文创候选  
→ 文创候选页  
→ 预览 / 下单（后续）

---

## 5. 每张流程图建议至少包含的内容

每张流程图不只画框和箭头，建议至少包含：

### 5.1 流程名称
例如：
- 孩子训练主链
- 家长候选设置流程
- 分享推广主链

### 5.2 参与角色
例如：
- 孩子
- 家长
- 系统
- 后台管理员

### 5.3 起点与终点
例如：
- 起点：首页
- 终点：作品进入展馆

### 5.4 关键判断点
例如：
- 是否上传成功
- 是否设置候选状态
- 是否达成有效激活
- 是否进入奖励记录

### 5.5 关键状态变化
例如：
- task_status 变化
- visibility_status 变化
- candidate_status 变化
- referral_status 变化
- reward_status 变化

---

## 6. 建议先画的 6 张流程图

如果设计资源有限，建议先画这 6 张：

1. 孩子训练主链  
2. 作品上传与入库主链  
3. 个人展馆与成长档案主链  
4. 家长作品管理主链  
5. 候选升级主链  
6. 分享推广主链  

这 6 张一旦成立，平台的主要逻辑就稳了。

## 7. 流程图命名建议

建议统一命名格式：

`流程序号_流程英文名_v1`

### 示例
- `01_child_training_flow_v1`
- `02_work_upload_flow_v1`
- `03_gallery_archive_flow_v1`
- `04_parent_management_flow_v1`
- `05_candidate_upgrade_flow_v1`
- `06_referral_reward_flow_v1`

## 8. 特别提醒：模块B“增加”动作要在流程中体现

模块B的特色不只在页面里，还要在任务与作品流程里体现。

建议在“模块B任务生成流程”或“作品元数据流程”中明确体现：

- 任务生成时可带 `increase`
- 作品上传后可写入 `increase_types_json`
- 作品详情页 / 后台可读取该信息

否则模块B的特色容易被埋掉。

---

## 9. 一句话总结

本项目的用户流程图，不是为了展示页面跳转，而是为了把这条真正重要的链画出来：

**训练如何变成作品，作品如何进入成长，家长如何管理路径，平台如何继续传播。**
