## 1. 文件说明

### 1.1 文件名称
`DEPLOYMENT_NOTES.md`

### 1.2 文件用途
用于定义平台战略首发版的：

- 部署前准备事项
- 环境与配置说明
- 初始化数据要求
- 上线开关策略
- 上线后观察点
- 问题排查建议

### 1.3 文件定位
这是上线前后团队必须共同查看的**部署与上线操作底稿**。

一句话说：

**代码能跑不代表能上线，这份文件负责把“能跑”变成“能发”。**

---

# 2. 上线目标说明

本项目首发版不是普通功能测试版，而是：

## **SMVP / 战略首发版**

因此上线目标不是“先扔出去试试”，而是：

- 首次体验要完整
- 核心链路要稳定
- 平台气质要成立
- 关键状态要可见
- 后台要可运营
- 开关要可控

---

# 3. 部署环境建议

建议至少准备三套环境：

## 3.1 本地开发环境
用于日常开发与联调。

## 3.2 测试环境 / 预发环境
用于：
- 产品验收
- 测试走查
- 内容初始化检查
- 首发演示

## 3.3 正式环境
用于真实用户试运行或首发上线。

# 6. 首发版功能开关建议

为了保证首发稳定，建议第一版用“开关式上线”。

---

## 6.1 建议默认开启
- [ ] 双训练模块
- [ ] 作品上传
- [ ] 我的展馆
- [ ] 成长档案简版
- [ ] 家长中心
- [ ] 分享推广中心简版

---

## 6.2 建议先做占位但不开重功能
- [ ] 文创候选入口可见，但真实下单关闭
- [ ] 存证候选入口可见，但第三方接入关闭
- [ ] 确权候选入口可见，但真实服务关闭
- [ ] 公共展示入口可见，但公开审核流可先关闭

---

## 6.3 建议严格控制的开关
- [ ] 邀请奖励实时发放
- [ ] 首单奖励自动生效
- [ ] 净收益结算显示
- [ ] 公共展示开放申请

这些建议首发期先偏保守。

# 9. 首发异常问题优先级建议

---

## P0（必须立即处理）
- 登录失败
- 任务无法生成
- 打印页打不开
- 上传无法完成
- 作品入库失败
- 家长中心打不开
- 邀请关系写错
- 奖励记录严重错乱

---

## P1（应尽快修复）
- 模块文案错
- 作品状态不同步
- 展馆筛选异常
- 候选状态显示错误
- 分享推广规则文案不清

---

## P2（可迭代优化）
- 页面细节问题
- 空状态文案优化
- 卡片样式优化
- 轻微性能优化

# 11. 首发阶段建议的运营策略

为了让首发版更稳，建议：

## 11.1 先用小范围测试用户
不要一开始大规模开放。

## 11.2 先让内容足够丰富
首发前模板不足，会直接伤害体验。

## 11.3 分享推广先保守
奖励规则先简单透明，不要一开始做太复杂。

## 11.4 候选系统先“能看见”
先让用户感受到未来路径，而不是急着全部打通。

**战略首发版的部署重点，不只是“服务上线”，而是确保双训练、作品、展示、家长、候选、传播六大系统上线后都是真实可感知、可运营、可继续长大的。**
