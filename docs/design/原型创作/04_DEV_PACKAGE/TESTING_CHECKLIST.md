# 2. 测试总原则

本项目测试不能只看“页面能不能打开”，必须同时看四层：

## 2.1 功能能不能用
按钮、接口、跳转、保存是否正常。

## 2.2 流程能不能走通
用户从入口到结果的整条链路是否断裂。

## 2.3 状态能不能一致
前台状态、后台状态、数据库状态是否一致。

## 2.4 产品表达是否成立
测试时也要判断：  
它是不是已经像一个“平台”，而不是功能拼接。
