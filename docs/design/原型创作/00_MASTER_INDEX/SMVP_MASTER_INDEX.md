## 1. 文件说明

### 1.1 文件名称
`SMVP_MASTER_INDEX.md`

### 1.2 文件用途
用于统一管理本项目当前阶段的全部核心文档，包括：

- 白皮书系统
- PRD 系统
- SMVP 系统
- 开发包系统
- 规则系统
- 后台系统
- 未来接口预留系统

### 1.3 文件定位
这是整个项目当前阶段的：

**总目录、总索引、总交接清单**

一句话说：

**以后任何人接手这个项目，先看这份。**

---

## 2. 项目当前总定义

本项目当前阶段，不再按普通 MVP 逻辑推进，而按：

## **SMVP（Strategic Minimum Viable Product）**
即：

**战略型最小可发布版**

来组织全部工作。

这意味着项目第一版追求的不是“最少功能”，而是：

- 最小但完整的战略表达
- 最小但完整的平台骨架
- 最小但清晰的未来接口布局
